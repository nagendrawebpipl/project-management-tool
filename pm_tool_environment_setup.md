# PM Tool MVP — Environment Setup (Docker + Supabase)

---

## Overview

Two environments, both using Docker-based Supabase:

| Environment | Supabase | App | Purpose |
|-------------|----------|-----|---------|
| **Local Dev** | Docker containers via Supabase CLI | `pnpm dev` (localhost:3000) | Daily development, testing, debugging |
| **Production** | Supabase Cloud (hosted) | Vercel | Live deployment |

---

## Environment 1: Local Development (Docker)

### Prerequisites

- Docker Desktop installed and running
- Node.js 18+ and pnpm installed
- Supabase CLI installed (`pnpm add -D supabase`)

### Initial Setup

```bash
# 1. Initialize Supabase in the project
pnpm supabase init

# 2. Start local Supabase (spins up Docker containers)
pnpm supabase start
```

This starts the following Docker containers:

| Container | Port | Purpose |
|-----------|------|---------|
| supabase-db | 54322 | PostgreSQL database |
| supabase-auth | 54321 | GoTrue auth server |
| supabase-rest | 54321 | PostgREST API |
| supabase-realtime | 54321 | Realtime server |
| supabase-storage | 54321 | Storage API |
| supabase-studio | 54323 | Supabase Studio (DB GUI) |
| supabase-inbucket | 54324 | Email testing (catches all emails) |
| supabase-kong | 54321 | API Gateway |

### Local URLs

After `supabase start`, you'll see output with these URLs:

```
API URL:        http://127.0.0.1:54321
GraphQL URL:    http://127.0.0.1:54321/graphql/v1
S3 Storage URL: http://127.0.0.1:54321/storage/v1/s3
DB URL:         postgresql://postgres:postgres@127.0.0.1:54322/postgres
Studio URL:     http://127.0.0.1:54323
Inbucket URL:   http://127.0.0.1:54324
anon key:       <auto-generated>
service_role key: <auto-generated>
```

### Local Environment File

**`.env.local`** (Git-ignored, local dev only):

```env
# ============================================
# LOCAL DEVELOPMENT ENVIRONMENT
# ============================================

# Supabase Local (Docker)
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon-key-from-supabase-start-output>
SUPABASE_SERVICE_ROLE_KEY=<service-role-key-from-supabase-start-output>

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
NODE_ENV=development
```

### Local Dev Commands

```bash
# Start Supabase Docker containers
pnpm supabase start

# Stop Supabase containers (preserves data)
pnpm supabase stop

# Stop and reset all data
pnpm supabase stop --no-backup

# View container status
pnpm supabase status

# Apply migrations to local DB
pnpm supabase db push

# Reset local DB (drop all + re-apply migrations)
pnpm supabase db reset

# Generate TypeScript types from local DB
pnpm supabase gen types typescript --local > src/types/database.types.ts

# Open Supabase Studio (DB GUI)
# Visit http://127.0.0.1:54323

# View emails (signup confirmation, password reset)
# Visit http://127.0.0.1:54324 (Inbucket)

# Start Next.js dev server
pnpm dev
```

### Local Database Access

```bash
# Direct psql connection
psql postgresql://postgres:postgres@127.0.0.1:54322/postgres

# Or via Supabase CLI
pnpm supabase db shell
```

## Environment 2: Production (Supabase Cloud + Vercel)

### Supabase Cloud Setup

1. Go to [supabase.com](https://supabase.com) and create a new project
2. Note down from Project Settings > API:
   - Project URL (`https://xxxx.supabase.co`)
   - `anon` public key
   - `service_role` key (keep secret)

### Push Migrations to Production

```bash
# Link to remote Supabase project
pnpm supabase link --project-ref <your-project-ref>

# Push all migrations to production
pnpm supabase db push

# Generate types from production (optional, should match local)
pnpm supabase gen types typescript --linked > src/types/database.types.ts
```

### Production Environment File

**`.env.production`** (for reference — actual values set in Vercel dashboard):

```env
# ============================================
# PRODUCTION ENVIRONMENT
# ============================================

# Supabase Cloud
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=<production-anon-key>
SUPABASE_SERVICE_ROLE_KEY=<production-service-role-key>

# App
NEXT_PUBLIC_APP_URL=https://your-domain.vercel.app
NODE_ENV=production
```

### Vercel Setup

1. Import repo to Vercel
2. Set environment variables in Vercel dashboard (Settings > Environment Variables):

| Variable | Value | Environment |
|----------|-------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://xxx.supabase.co` | Production |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `eyJ...` | Production |
| `SUPABASE_SERVICE_ROLE_KEY` | `eyJ...` | Production |
| `NEXT_PUBLIC_APP_URL` | `https://your-domain.vercel.app` | Production |

3. Configure Supabase Auth:
   - Go to Supabase Dashboard > Authentication > URL Configuration
   - Set **Site URL** to your production URL
   - Add redirect URLs:
     - `https://your-domain.vercel.app/auth/callback`
     - `http://localhost:3000/auth/callback` (for local dev)

### Production Storage Setup

In Supabase Dashboard > Storage:
- Verify `task-attachments` bucket exists (created by migration)
- If not, create it manually as a private bucket

### Production Realtime Setup

In Supabase Dashboard > Database > Replication:
- Enable realtime for tables: `tasks`, `task_comments`, `notifications`

## Supabase Config File

**`supabase/config.toml`** — Customize local Docker Supabase:

```toml
[project]
id = "pm-tool"

[api]
enabled = true
port = 54321
schemas = ["public", "storage"]
extra_search_path = ["public", "extensions"]
max_rows = 1000

[db]
port = 54322
shadow_port = 54320
major_version = 15

[db.pooler]
enabled = false
port = 54329
pool_mode = "transaction"
default_pool_size = 20
max_client_conn = 100

[studio]
enabled = true
port = 54323
api_url = "http://127.0.0.1"

[inbucket]
enabled = true
port = 54324
smtp_port = 54325
pop3_port = 54326

[storage]
enabled = true
file_size_limit = "10MiB"

[auth]
enabled = true
site_url = "http://localhost:3000"
additional_redirect_urls = ["http://localhost:3000/auth/callback"]
jwt_expiry = 3600
enable_refresh_token_rotation = true
refresh_token_reuse_interval = 10
enable_signup = true
enable_anonymous_sign_ins = false

[auth.email]
enable_signup = true
double_confirm_changes = true
enable_confirmations = false  # Set to false for easier local dev (no email confirmation needed)

[auth.sms]
enable_signup = false
enable_confirmations = false

[realtime]
enabled = true
```

**Key local dev setting:** `enable_confirmations = false` means new signups work instantly without email confirmation. In production, Supabase Cloud handles this via their email service.

## Daily Development Workflow

```bash
# 1. Start Docker containers (if not already running)
pnpm supabase:start

# 2. Start Next.js dev server
pnpm dev

# 3. Work on features...

# 4. If you modify migrations:
pnpm supabase:reset     # Reset DB and re-apply all migrations
pnpm supabase:types     # Regenerate TypeScript types

# 5. When done:
pnpm supabase:stop       # Stop containers (data preserved)
```

---

## Deployment Workflow

```bash
# 1. Ensure local build works
pnpm build

# 2. Push migrations to production
pnpm supabase link --project-ref <ref>
pnpm supabase db push

# 3. Push code to main branch
git push origin main

# 4. Vercel auto-deploys from main
# 5. Verify production at your Vercel URL
```

## Troubleshooting

### Docker Issues

```bash
# Containers not starting
docker ps                          # Check running containers
docker logs supabase-db            # Check DB logs
pnpm supabase stop --no-backup     # Full reset
pnpm supabase start                # Fresh start

# Port conflicts
# Edit supabase/config.toml to change ports if 54321-54326 are in use

# Docker disk space
docker system prune                # Clean unused containers/images
```

### Database Issues

```bash
# Migration failed
pnpm supabase db reset             # Reset and re-apply all migrations

# Check current migration status
pnpm supabase migration list

# Create a new migration
pnpm supabase migration new <name>
```

### Auth Issues

```bash
# Local: Emails not working
# Visit http://127.0.0.1:54324 (Inbucket) to see captured emails

# Production: Auth callback failing
# Check Supabase Dashboard > Auth > URL Configuration
# Ensure redirect URLs include your production domain
```
